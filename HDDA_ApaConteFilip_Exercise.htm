Descrizione del Problema

Il Dataset Splice contiene 400 siti di splice di donatori umani con una sequenza di 7 basi. La variabile binaria y indica se la i-esima combinazione corrisponde o meno ad un sito di splice (codificato con 1 in caso affermativo e 0 viceversa). Il nostro obbiettivo quell di costruire un modello che sia capace di rilevare se una data combinazione di basi in una delle 7 posizioni corrisponde ad un sito di splice.

Come primo passo importiamo le librerie necessarie allanalisi e guardiamo la struttura dei dati.

Per impostare correttamente il problema di regressione, dato che abbiamo a che fare con dati categorici, andiamo a definire la lista dei contrasti. Ogni contrasto corrisponde ad una delle sette posizioni sul filamento di DNA e ognuno dei 4 livelli di ogni contrasto consiste nella singola base azotata.

Definizione Contrasti

# Definiamo una lista contenente i contrasti dei fattori creati
contr <- rep(list("contr.treatment", base=NULL), ncol(splice)-1, )
names(contr) <- names(splice)[-1]

Andiamo ora a splittare i dati in Training set e Test Set. Operiamo un campionamento 80%-20% stratificato sulla variabile di risposta y.

Splitting dei Dati

# Splitting dati in Training e Test Set (80-20)

set.seed(1)
sample=sample.split(splice$y,SplitRatio = 0.80)
train_set=subset(splice,sample==TRUE)
test_set=subset(splice,sample==FALSE)

Fitting del Modello

Impostiamo il modello di Regressione Logistica con tecnica Group Lasso al fine di operare, contestualmente al fitting, anche uno shrinkage dei coefficienti.

logmodel <- grplasso(y ~ ., data = train_set, model = LogReg(), lambda = seq(15, 0, by= -1),
                       contrasts = contr, center = TRUE, standardize = TRUE)

Coefficient path

Visualizziamo ora il sentiero dei coefficienti, ovvero vediamo come cambiano i valori dei coefficienti dei vari gruppi in funzione del valore decrescente di lambda. Possiamo vedere come, allaumentare di lambda, tutti i coefficienti si avvicinino allo 0.

Predizioni

Le predizioni di un modello di Regressione Logistica sono codificate tra i valori [0,1]. Qui, plottiamo il valore delle predizioni fatte da 5 modelli con diversi valori di lambda sullo stesso Test Set. Il nostro obiettivo, in questo caso, quello di vedere il grado di incertezza di ogni modello. Ovvero, quante delle predizioni del modello sono pi vicine al valore medio tra 0 e 1 piuttosto che ad uno degli estremi.

# Procedura di visualizzazione per valori multipli di lambda.

Scoring del Modello Migliore

Il modello meno insicuro sembra essere quello con lambda = 15. Vediamo la sua performance in termini di curva ROC, Area sotto la Curva e Accuracy (settiamo un valore soglia di 0.75 per le predizioni effettuate sul test set).

#  Curva ROC Penalizzata
par(mfrow=c(1,1))
predictTrain <- predict(logmodel, type = 'response')
ROCRpred_penalizzata = prediction(predictTrain[, 16], train_set$y)
ROCRperf_penalizzata= performance(ROCRpred_penalizzata,'tpr','fpr')
plot(ROCRperf_penalizzata, colorize=TRUE, print.cutoffs.at=seq(0,1,by=0.1),
     text.adj=c(-0.2,2), main = "Regressione Group Lasso con lambda = 15")

# Area sotto la curva
auc_penalizzata = as.numeric(performance(ROCRpred_penalizzata, 'auc')@y.values)
# Accuracy con valore-soglia di 0.75
predictTest = predict(logmodel, newdata = test_set ,type = "response")
conf_matrix <- table(test_set$y, predictTest[, 16] >= 0.8)               # Accuracy del modello con lambda = 15
conf_matrix
##

accuracy_penalizzata <- (38+36)/(38+36+4+2)
